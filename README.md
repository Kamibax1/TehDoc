# Высокопроизводительный Асинхронный Сервис на Kotlin с Корутинами

Данный проект представляет собой высокопроизводительный асинхронный сервис на Kotlin с использованием корутин. Он оптимизирован для высокой пропускной способности в микросервисах. Ключевой аспект — безопасный доступ к общим ресурсам для предотвращения состояний гонки. Проект углубляет знания в асинхронном программировании, демонстрируя интеграцию корутин с механизмами синхронизации (мьютексы/акторы). Включает комплексную обработку ошибок для создания масштабируемого, отказоустойчивого решения под высокой нагрузкой.

Для запуска проекта нужно клонировать проект.
Откройте терминал и введите команды:
```
cd <Имя папки вашего проекта>
```
```
git clone https://github.com/Kamibax1/TehDoc
```
Зайдите в IDE IntelliJ IDEA.
Откройте файл src/main/kotlin/Main.kt.
Нажмите зеленую кнопку "Run" рядом с функцией main().

## Технические требования
Проект разработан с использованием языка Kotlin и системы сборки Gradle. Для успешного запуска и работы необходимы следующие компоненты и версии:
| Требование | Значение | Описание |
|---|---|---|
| **Язык программирования** | Kotlin | Основной язык разработки проекта. |
| **Среда разработки (IDE)** | IntelliJ IDEA | Рекомендуется версия Community Edition. |
| **Система сборки** | Gradle - Kotlin DSL | Используется для управления зависимостями и сборки проекта. |
| **Основная библиотека** | `kotlinx-coroutines-core:1.7.3` | Ключевая библиотека для реализации асинхронности и конкурентного доступа. |

# Тестирование и обратная связь

Проведено комплексное тестирование сервиса ConcurrentService с участием 10 тестировщиков. 

Тестирование включало:

- **Интеграционное тестирование** - работа в конкурентной среде  
- **Пользовательское тестирование** - оценка удобства API

Все отчеты о тестировании находятся в папке [`/testing`](./testing/):

На основе обратной связи реализованы:

2. **Добавлен метод сброса** - resetCounter() для обнуления счетчика
3. **Оптимизированы задержки** - уменьшено время ответа сервиса
